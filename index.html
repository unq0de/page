## who I am

        
I'm a developer who likes seeing the whole picture — from the screen someone taps on to the API request that makes it actually do something. Most of my time goes into mobile and web apps built with React Native, Flutter and React, backed by Python APIs and backends that hold up under real use.


        
## how I work

        
    
          
  • Start from how it should feel to use, not just how it should be built
    • 
          
  • Keep the code as clean as the interface it powers
    • 
          
  • Ship early, then refine with real feedback
    • 
          
  • Ask questions before making assumptions
    • 
        


        
## currently

        
Digging into on-device ML for mobile apps, and getting faster at turning rough sketches into working prototypes.


        
## off the clock
